# 模板引擎能力对标

> 对比对象：`Jinja2`、`Django Template Language`、`Mako`、`Python Liquid`
> 更新日期：2026-06-12
> 用途：为 `mold` 的项目路线和设计决策提供外部参照，不作为调用方文档。

---

## 一、当前 mold 已实现的能力

- 变量插值 `{{ expr }}`
- 点路径访问 `{{ user.name }}`
- 条件分支 `{% if expr %} ... {% else %} ... {% endif %}`
- 循环 `{% for item in items %} ... {% endfor %}`
- filter 链 `{{ expr | upper | trim }}`
- filter 参数（字符串、整数、路径变量）
- 自定义 filter（通过 `Engine::register_filter`，不可覆盖内置 filter）
- 比较表达式 `== != < <= > >=`
- 布尔表达式 `and or not`
- 括号分组
- 手动 `escape`（`| escape`，显式使用，不自动转义）
- `safe` filter 与 Engine 级 autoescape
- `{% include "name" %}` 与 `Loader`
- whitespace control：`{%-` / `-%}` / `{{-` / `-}}`
- `{# ... #}` 注释
- `length` filter
- 结构化错误与源码位置 `SourceSpan`
- `Template::parse` / `Template::render` 分层（parse once, render many）
- `Template::ast()` 调试访问
- `from_json(...)` / `from_map(...)` 上下文转换
- `Engine` 层扩展点
- WASM 导出 `mold_render(...)`
- 8 个示例程序，105 个测试，benchmark

## 三、逐项目重合度判断

### Jinja2

- 当前重合度：**中等**
- 已重合：插值、点路径、`if/for`、filter、比较/布尔、手动 escape、include、loader、autoescape、whitespace control、注释、parse/render 分层
- 未重合：模板继承、import、macro/call、tests、undefined 策略、sandbox、async、大量内置 filters
- 判断：`mold` 已覆盖 Jinja 的核心骨架和一部分工程化能力，但仍应保持轻量边界

### Django Template Language

- 当前重合度：**中等偏低～中等**
- 已重合：插值、点路径、`if/for`、filter、escape、include、模板偏展示层的克制路线
- 未重合：extends、block、partials、autoescape 默认开启、自定义 tag library、更完整的标签体系
- 判断：设计哲学与 `mold` 接近，但 Django 在「页面组合」和「HTML 安全默认值」上更成熟

### Mako

- 当前重合度：**低～中等**
- 已重合：插值、条件/循环、filter、预编译/复用模板
- 显著差异：Mako 强调 Python 代码嵌入，有 defs、blocks、namespaces、inheritance、caching，更像「模板形式的 Python」
- 判断：`mold` 不应以 Mako 为主要追赶对象，路线不同

### Python Liquid

- 当前重合度：**高**
- 已重合：customer-facing / safe-ish 定位更接近；插值、条件、循环、filters；parse/render 分层；Environment/Engine 风格扩展点；自定义 filters；loader；include；autoescape 配置
- 未重合：custom tags、static analysis、i18n、Liquid 风格的 render/include 细分语义
- 判断：路线相似度最高，都偏「受控模板语言」而非「可执行模板脚本」

### 当前重合度变化（include + loader + autoescape + whitespace control 已落地后）

- Jinja2：中等偏高
- Django：中等
- Python Liquid：高
- Mako：仍为中等以下（不以此为追赶目标）

## 四、路线参考建议

| 维度 | 参考对象 |
|---|---|
| 语法骨架 | Jinja2 |
| 边界克制 | Django Template Language |
| 安全与 environment 设计 | Python Liquid |
| 不应盲目追 | Mako（可执行模板脚本路线） |

`mold` 的最佳定位不是「MoonBit 版 Jinja2 全家桶」，而是：

- 核心表达力像 Jinja
- 边界感像 Django
- 环境/安全策略像 Liquid
- 不走 Mako 那种可执行模板脚本路线

## 六、场景定位

`mold` 同时支持通用文本和 HTML，但当前优先面向通用文本场景：

- 报告生成
- 邮件模板
- 配置文件生成
- 文档模板

HTML 输出通过显式 `| escape`、Engine 级 `with_autoescape(true)` 与 `| safe` 支持，不把 HTML 安全策略作为默认负担。
